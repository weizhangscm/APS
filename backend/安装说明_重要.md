# ⚠️ 重要：后端依赖安装问题解决方案

## 问题根源

您的系统中设置了用户级别的环境变量 `PIP_NO_INDEX=1`，这会**完全阻止** pip 从任何索引（包括镜像源）下载包。

这就是为什么所有安装尝试都显示 "from versions: none" 的原因。

## 解决方案

### 方法 1：删除环境变量（推荐）

**步骤 1：删除 PIP_NO_INDEX 环境变量**

1. 按 `Win + R`，输入 `sysdm.cpl`，回车
2. 点击"高级"选项卡
3. 点击"环境变量"按钮
4. 在"用户变量"部分，找到 `PIP_NO_INDEX`
5. 选中它，点击"删除"
6. 点击"确定"保存

**或者使用提供的批处理文件（需要管理员权限）：**

右键点击 `删除PIP_NO_INDEX.bat`，选择"以管理员身份运行"

**步骤 2：重新打开 PowerShell**

关闭当前的 PowerShell 窗口，重新打开一个新的窗口。

**步骤 3：安装依赖**

```powershell
cd C:\Users\Chun.Wang\Desktop\APS\backend
python -m pip install --index-url https://mirrors.aliyun.com/pypi/simple/ --trusted-host mirrors.aliyun.com -r requirements.txt
```

### 方法 2：临时解决方案（不推荐）

如果您暂时无法删除环境变量，可以尝试：

1. 以管理员身份运行 PowerShell
2. 执行以下命令：

```powershell
cd C:\Users\Chun.Wang\Desktop\APS\backend
[System.Environment]::SetEnvironmentVariable("PIP_NO_INDEX", $null, "User")
```

然后重新打开 PowerShell 窗口。

### 方法 3：使用虚拟环境（如果上述方法都不行）

创建虚拟环境可以绕过某些环境变量限制：

```powershell
cd C:\Users\Chun.Wang\Desktop\APS\backend
python -m venv venv
.\venv\Scripts\Activate.ps1
python -m pip install --index-url https://mirrors.aliyun.com/pypi/simple/ --trusted-host mirrors.aliyun.com -r requirements.txt
```

## 验证安装

安装完成后，运行：

```powershell
python -c "import fastapi, uvicorn, sqlalchemy, pydantic, dateutil, aiosqlite; print('所有依赖已成功安装！')"
```

如果看到 "所有依赖已成功安装！"，说明安装成功。
