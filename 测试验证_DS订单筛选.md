# 测试验证：DS订单筛选功能

## 测试场景

### 场景1：筛选"待排程"订单

**操作步骤**：
1. 打开浏览器，访问 `http://localhost:5173`
2. 进入"DS订单数据"页面
3. 在"状态筛选"下拉框中选择"待排程"

**预期结果**：
- 显示所有工序状态为 `pending` 的计划订单
- 这些订单在"状态"列显示为"待排程"（蓝色标签）
- 大约会显示 30-40 个计划订单

**验证点**：
- ✓ 有订单显示出来（不是空白）
- ✓ 所有显示的订单状态标签都是"待排程"
- ✓ 所有显示的订单类型标签都是"计划订单"（蓝色）

### 场景2：筛选"已排程"订单

**操作步骤**：
1. 在"状态筛选"下拉框中选择"已排程"

**预期结果**：
- 显示所有生产订单（15个）
- 显示工序已全部排程的计划订单（如果有）
- 这些订单在"状态"列显示为"已排程"（蓝色标签）

**验证点**：
- ✓ 有订单显示出来
- ✓ 所有显示的订单状态标签都是"已排程"
- ✓ 应该能看到生产订单（灰色标签：PRD开头的订单号）

### 场景3：组合筛选 - 订单类型 + 状态

**操作步骤**：
1. 在"订单类型"下拉框选择"计划订单"
2. 在"状态筛选"下拉框选择"待排程"

**预期结果**：
- 只显示计划订单
- 只显示待排程的订单
- 不显示生产订单

**验证点**：
- ✓ 所有订单类型都是"计划订单"
- ✓ 所有订单状态都是"待排程"

### 场景4：组合筛选 - 订单类型 + 状态（生产订单）

**操作步骤**：
1. 清除所有筛选
2. 在"订单类型"下拉框选择"生产订单"
3. 在"状态筛选"下拉框选择"已排程"

**预期结果**：
- 只显示生产订单（15个）
- 所有订单状态都是"已排程"
- 订单号都是 PRD 开头

**验证点**：
- ✓ 显示 15 个订单
- ✓ 所有订单类型都是"生产订单"（灰色标签）
- ✓ 所有订单状态都是"已排程"

### 场景5：清除筛选

**操作步骤**：
1. 点击"状态筛选"和"订单类型"的清除按钮（×）

**预期结果**：
- 显示所有订单（57个：42个计划订单 + 15个生产订单）
- 可以看到不同状态的订单

**验证点**：
- ✓ 显示的订单数量约为 57 个
- ✓ 可以看到"计划订单"和"生产订单"
- ✓ 可以看到"待排程"和"已排程"状态

## 快速验证方法

在浏览器控制台执行：

```javascript
// 1. 检查当前显示的订单数据
const store = window.$app?.config?.globalProperties?.$pinia?.state?.value?.dsFilters
console.log('总订单数:', store?.dsOrders?.length)

// 2. 统计订单类型和状态
const orders = store?.dsOrders || []
const planned = orders.filter(o => o.order_type === 'planned').length
const production = orders.filter(o => o.order_type === 'production').length
console.log('计划订单:', planned, '生产订单:', production)

// 3. 检查工序状态分布
const ordersWithPendingOps = orders.filter(o => 
  o.operations?.some(op => op.status === 'pending')
).length
console.log('有待排程工序的订单:', ordersWithPendingOps)
```

## 问题排查

如果筛选"待排程"仍然没有结果：

1. **检查数据是否加载**
   - 打开浏览器开发者工具 -> Network 标签
   - 刷新页面，查看是否有 `/api/orders` 请求
   - 检查响应数据中是否有订单

2. **检查工序数据**
   - 点击任意订单的"工序"按钮
   - 查看工序状态是否为"待排程"
   - 如果所有工序都是"已排程"，说明数据已经全部排程了

3. **检查控制台错误**
   - 打开浏览器开发者工具 -> Console 标签
   - 查看是否有 JavaScript 错误

## 数据状态说明

根据当前数据库检查：

| 订单类型 | 数量 | 订单状态 | 工序状态 |
|---------|------|----------|---------|
| 计划订单 | 42 | scheduled | pending |
| 生产订单 | 15 | scheduled | scheduled |

这意味着：
- 筛选"待排程"：应显示 42 个计划订单
- 筛选"已排程"：应显示 15 个生产订单（+ 可能的已排程计划订单）

## 修复前 vs 修复后

### 修复前
- 筛选"待排程"：0 个结果（因为数据库中没有 status='created' 的订单）
- 筛选"已排程"：所有 57 个订单

### 修复后
- 筛选"待排程"：约 42 个计划订单（工序状态为 pending）
- 筛选"已排程"：约 15 个生产订单（+ 已排程的计划订单）
