## 1. 系统概述

### 1.1 系统简介

AI Native APS（Advanced Planning and Scheduling）是一个参考 SAP PPDS 设计的工序级高级计划排程系统。系统支持甘特图可视化、有限产能排程、拖拽调整和 KPI 仪表板，帮助制造企业优化生产计划、提高资源利用率、确保订单准时交付。

### 1.2 核心价值

- **可视化排程**：通过甘特图直观展示生产计划，支持拖拽调整
- **智能排程**：支持正向/逆向排程，多种优先级规则
- **有限产能**：考虑资源实际产能进行排程，避免超负荷
- **实时监控**：KPI仪表板实时展示关键指标
- **约束检测**：自动检测资源冲突、工序顺序等约束违反

### 1.3 应用场景

- 离散制造业生产计划排程
- 多品种、小批量生产环境
- 需要精细化工序级排程的场景
- 需要实时监控生产进度和资源利用的企业

## 2. 功能模块说明

### 2.1 详细计划表

**菜单路径**: 详细计划表

详细计划表是系统的核心功能模块，提供工序级别的排程展示和操作。通过可视化的甘特图界面，用户可以直观地查看生产计划、执行自动排程、手动调整工序时间，并实时监控资源利用情况。

#### 2.1.1 界面布局

详细计划表页面分为两个主要区域：

1. **工具栏区域**（页面顶部）
   - 第一行：筛选条件区域（显示区间、资源筛选、产品筛选）
   - 第二行：操作控制区域（图表选择、缩放控制、操作按钮、图例）

2. **图表区域**（页面主体）
   - 可同时显示多个图表（资源图表、产品图表、资源利用率图表）
   - 图表之间有可拖动的分隔条，支持调整高度比例

#### 2.1.2 筛选功能

| 筛选项 | 说明 | 默认值 |
|--------|------|--------|
| 显示区间 | 日期范围选择器，决定甘特图显示的时间范围 | 前5天到未来1个月 |
| 资源名称 | 多选下拉框，筛选要显示的资源 | 空（需用户选择） |
| 产品 | 多选下拉框，筛选要显示的产品 | 空（需用户选择） |

**显示区间快捷选项**：

| 快捷选项 | 说明 |
|----------|------|
| 前5天到未来1月 | 从当前日期前5天到未来1个月（默认） |
| 本周 | 当前周的周日到周六 |
| 本月 | 当前月的第一天到最后一天 |
| 最近30天 | 从30天前到当前日期 |

**资源筛选特性**：

- **全选**：快速选择所有资源
- **瓶颈**：仅显示标记为瓶颈的资源（默认勾选）
- 支持多选和折叠标签显示

**产品筛选特性**：

- **全选**：快速选择所有产品
- 支持多选和折叠标签显示

**筛选条件持久化**：

- 筛选条件自动保存到浏览器 localStorage
- 10分钟内有效，超时后恢复默认值
- 包括：日期范围、已选资源、已选产品、图表选择、缩放级别、瓶颈筛选状态

#### 2.1.3 图表展示

**图表选择**：

| 图表类型 | 说明 |
|----------|------|
| 资源图表 | 以资源为第一维度展示工序（默认选中） |
| 产品图表 | 以产品为第一维度展示工序 |
| 资源利用图表 | 以柱状图形式展示各资源的利用率 |


**产品甘特图结构**：

```

└── ...
```

- 第一级显示产品名称
- 第二级显示该产品相关的订单工序

**资源利用率图**：

- 以柱状图形式展示每个时间槽的资源利用率
- 绿色柱子：正常利用率（0-100%）
- 红色柱子：超载（利用率 > 100%）
- 左侧显示资源名称和产能刻度

**图例说明**：

| 颜色 | 含义 |
|------|------|
| 蓝色 | 计划订单的工序 |
| 灰色 | 生产订单的工序 |

**当前时间线**：

- 红色竖线标记当前时间在甘特图中的位置
- 实时更新，便于识别当前进度

| 按钮 | 功能 | 说明 |
|------|------|------|
| 放大/缩小 | 调整缩放级别 | 点击切换到更精细/更宏观的视图 |

| 警报 | 查看警报信息 | 下拉菜单包含"订单延误"和"物料短缺"两类警报 |
| 策略 | 配置排程策略 | 打开策略配置对话框 |
| 取消计划 | 清除排程 | 清除选中资源/产品上的排程，工序恢复为待排程状态 |
| 自动计划 | 执行排程 | 下拉菜单包含"启发式"和"优化器"两种排程方式 |

| 日志 | 查看排程日志 | 显示最近一次排程操作的详细结果和错误信息 |

#### 2.1.5 策略配置

点击"策略"按钮打开策略配置对话框，可配置以下参数：

| 参数 | 说明 | 可选值 |
|------|------|--------|
| 排序规则 | 决定订单的排程顺序 | 订单优先级 |
| 计划模式 | 排程时如何查找可用时段 | 查找槽位（有限产能） |
| 计划方向 | 排程的搜索方向 | 向前（从期望日期向未来）、向后（从交期向过去） |
| 期望日期 | 排程的起点日期 | 当前日期、指定日期 |
| 订单内部关系 | 是否考虑同一订单内工序的时间关系 | 不考虑、始终考虑 |
| 子计划模式 | 关联工序的排程方式 | 根据调度模式调度相关操作、以无限方式调度相关操作 |
| 计划出错的操作 | 工序无法排程时的处理方式 | 立即终止 |

**参数详解**：

- **向前排程**：从期望日期（当前日期或指定日期）开始，向未来搜索可用时段，适合尽早开始生产的场景
- **向后排程**：从交货期开始，向过去搜索可用时段，适合减少库存积压的JIT生产场景
- **订单内部关系**：
  - "不考虑"：只排选中资源上的工序，不调整其他工序
  - "始终考虑"：自动调整订单内其他工序以维护工序顺序约束

#### 2.1.6 自动排程

**启发式排程**：

点击"自动计划" > "启发式"，打开启发式配置对话框：

| 启发式算法 | 说明 |
|------------|------|
| 稳定向前计划 | 基于策略配置执行正向排程，优先保持已有排程不变，只排程未排程的工序 |

执行启发式排程时：
1. 必须先选择要排程的资源
2. 系统根据策略配置的排序规则对订单排序
3. 按顺序为每个订单的工序查找可用时段
4. 考虑资源产能约束和工序顺序约束
5. 排程结果先存入缓存（预览模式），不直接写入数据库

**优化器排程**：

点击"自动计划" > "优化器"，打开优化器配置对话框：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 延迟成本权重 | 订单延误的惩罚权重 | 30 |
| 库存成本权重 | 提前完工导致库存的惩罚权重 | 20 |
| 切换成本权重 | 产品切换的惩罚权重 | 25 |
| 产能利用权重 | 资源利用率的优化权重 | 25 |
| 优化时间限制 | 优化器运行的最大时间（秒） | 60 |

优化器会在给定的时间限制内，综合考虑各项成本权重，搜索最优的排程方案。

#### 2.1.7 排程缓存机制

系统采用"预览-确认"的两阶段排程机制：

**预览阶段**：
1. 执行自动排程后，结果存入服务器内存缓存
2. 甘特图显示排程预览结果
3. 此时数据库中的数据不变
4. 工具栏显示"丢弃计划"按钮，"保存计划"按钮高亮显示

**确认阶段**：
- **保存计划**：将缓存中的排程结果写入数据库，更新订单和工序状态为"已排程"
- **丢弃计划**：清空缓存，恢复到排程前的状态

**设计意图**：
- 用户可以在保存前预览排程效果
- 如果排程结果不满意，可以调整参数重新排程或直接丢弃
- 避免误操作导致的数据污染

#### 2.1.8 取消计划

"取消计划"功能用于清除已有的排程：

1. 根据选中的资源和/或产品筛选工序
2. 清除这些工序的排程时间（scheduled_start、scheduled_end）
3. 将工序状态恢复为"待排程"（pending）
4. 同步更新订单状态

**使用场景**：
- 排程结果不满意，需要重新排程
- 业务调整，需要取消部分资源的排程

#### 2.1.9 警报功能

点击"警报"下拉按钮查看警报信息：

| 警报类型 | 说明 |
|----------|------|
| 订单延误 | 显示预计无法按时交货的订单列表 |
| 物料短缺 | 显示物料不足的订单列表 |

警报详情表格显示：
- 订单号
- 产品名称
- 描述（延误天数或短缺物料）
- 严重程度（高/中/低）

#### 2.1.10 排程日志

点击"日志"按钮查看最近一次排程操作的详细结果：

| 信息 | 说明 |
|------|------|
| 排程结果 | 成功或失败的消息 |
| 时间戳 | 排程操作的执行时间 |
| 统计信息 | 排程的订单数和工序数 |
| 详情列表 | 每个订单的排程结果（成功/失败、工序数、错误信息） |

**用途**：
- 了解排程执行的详细情况
- 排查排程失败的原因
- 确认哪些订单被成功排程

### 2.2 KPI仪表板

**菜单路径**: KPI仪表板

KPI仪表板提供关键生产指标的可视化展示，帮助管理者实时监控生产状况、资源利用率和订单交付情况。

#### 2.2.1 界面布局

KPI仪表板页面分为四个主要区域：

1. **页面头部**
   - 标题："KPI仪表板"
   - "刷新数据"按钮：手动刷新所有KPI数据

2. **KPI指标卡片区**（顶部横向排列）
   - 4个指标卡片等宽排列
   - 每个卡片包含：图标、指标名称、数值、单位

3. **图表区**（中部）
   - 左侧：资源利用率柱状图（占50%宽度）
   - 右侧：每日产能负荷图（占50%宽度）

4. **资源利用详情表**（底部）
   - 表格形式展示各资源的详细利用数据

#### 2.2.2 KPI指标卡片

| 指标 | 说明 | 计算方法 | 颜色规则 |
|------|------|----------|----------|
| 总订单数 | 系统中所有订单的数量 | 统计所有生产订单和计划订单 | 蓝色（固定） |
| 已排程订单 | 状态为"已排程"的订单数量 | 统计订单status='scheduled'的数量 | 绿色（固定） |
| 订单准时率 | 预计能准时交付的订单比例 | (准时订单数 / 总订单数) × 100% | 动态颜色 |
| 平均提前期 | 订单从开始到完成的平均时间跨度 | 所有订单提前期的平均值（小时） | 蓝色（固定） |

**订单准时率颜色编码**：

| 准时率范围 | 颜色 | 含义 |
|------------|------|------|
| >= 90% | 绿色 | 良好，大部分订单能准时交付 |
| 70% - 90% | 黄色 | 警告，存在一定延误风险 |
| < 70% | 红色 | 危险，需要立即关注 |

#### 2.2.3 订单准时率计算逻辑

```
对于每个订单：
1. 获取预计完工时间：
   - 优先使用：最后一道工序的 scheduled_end
   - 如果没有排程时间，使用订单的 confirmed_end
   - 如果仍没有，使用估算值（due_date）
2. 比较预计完工时间与交货期（due_date）
3. 如果 预计完工时间 ≤ 交货期，则为准时订单
4. 准时率 = 准时订单数 / 总订单数 × 100%
```

#### 2.2.4 平均提前期计算逻辑

```
对于每个订单：
1. 找出订单所有工序的时间范围：
   - 最早开始时间 = MIN(所有工序的 scheduled_start)
   - 最晚结束时间 = MAX(所有工序的 scheduled_end)
2. 提前期 = 最晚结束时间 - 最早开始时间（小时）
3. 如果订单未排程，使用估算提前期
4. 平均提前期 = 所有订单提前期之和 / 订单数量
```

#### 2.2.5 资源利用率图表

**图表类型**：柱状图

**数据结构**：
- X轴：资源名称
- Y轴：利用率百分比（0-100%）

**颜色编码**：

| 利用率范围 | 颜色 | 含义 |
|------------|------|------|
| >= 90% | 红色 | 过载，资源压力过大 |
| 70% - 90% | 黄色 | 高负荷，需关注 |
| < 70% | 绿色 | 正常，有剩余产能 |

**交互功能**：
- 鼠标悬停：显示资源名称和具体利用率数值
- 柱状图顶部圆角设计，符合 Material Design 3 风格

#### 2.2.6 每日产能负荷图

**图表类型**：组合图（柱状图 + 折线图）

**数据结构**：
- X轴：日期（未来14天，格式如 01-15、01-16）
- Y轴：产能（小时）

**图表元素**：

| 元素 | 类型 | 说明 |
|------|------|------|
| 已用产能 | 柱状图（蓝色） | 当天已排程的总工时 |
| 总产能 | 折线图（灰色虚线） | 当天可用的总产能上限 |

**交互功能**：
- 图例：点击可切换显示/隐藏对应数据系列
- 鼠标悬停：显示当天的已用产能和总产能数值

**产能负荷分析**：
- 当已用产能接近或超过总产能时，表示资源紧张
- 可用于识别产能瓶颈日期，提前进行资源调配

#### 2.2.7 资源利用详情表

**表格字段**：

| 字段 | 说明 |
|------|------|
| 资源名称 | 资源的显示名称 |
| 工作中心 | 资源所属的工作中心 |
| 总产能(小时) | 统计周期内资源的总可用产能 |
| 已排程(小时) | 统计周期内已排程的工时 |
| 利用率 | 进度条 + 百分比数值 |

**利用率进度条颜色**：
- 与资源利用率图表颜色编码一致
- 进度条长度表示百分比
- 超过100%时，进度条显示满格并标注实际百分比

**表格特性**：
- 斑马纹样式（隔行变色）
- 支持按列排序

##### 详情对话框（所有字段）

#### 2.3.5 切换矩阵管理

**菜单路径**: 主数据管理 > 切换矩阵

切换矩阵用于定义不同产品组之间切换时所需的换型时间。排程引擎在安排连续任务时，会考虑切换时间。

##### 界面布局（三个标签页）

1. **切换组标签页**
   - 管理切换组的创建和编辑

2. **产品分配标签页**
   - 将产品分配到切换组

3. **切换矩阵标签页**
   - 设置切换组之间的切换时间

##### 标签页一：切换组

用于创建和管理切换组。

| 字段 | 说明 | 必填 |
|------|------|------|
| 切换组代码 | 切换组的唯一标识 | 是 |
| 切换组名称 | 切换组的显示名称 | 是 |
| 描述 | 切换组的说明信息 | 否 |

**切换组示例**：
- GROUP_A: 小型零件组
- GROUP_B: 中型零件组
- GROUP_C: 大型零件组

##### 标签页二：产品分配

将产品分配到对应的切换组。

| 列 | 说明 |
|----|------|
| 产品代码 | 产品标识 |
| 产品名称 | 产品名称 |
| 当前切换组 | 已分配的切换组（可为空） |
| 操作 | 分配按钮 |

**操作方式**：
- 点击"分配"按钮，弹出切换组选择对话框
- 选择目标切换组后确认
- 一个产品只能属于一个切换组

##### 标签页三：切换矩阵

定义从一个切换组切换到另一个切换组所需的时间（分钟）。

**矩阵范围选择**：

| 范围 | 说明 | 优先级 |
|------|------|--------|
| 全局 | 适用于所有资源 | 最低 |
| 工作中心 | 适用于特定工作中心的所有资源 | 中 |
| 资源 | 适用于特定资源 | 最高 |

**矩阵编辑**：

| 特性 | 说明 |
|------|------|
| 行/列 | 行表示"从"切换组，列表示"到"切换组 |
| 对角线 | 始终为0（同组切换无需时间） |
| 其他单元格 | 可编辑，输入切换时间（分钟） |
| 非对称 | A→B 和 B→A 的切换时间可以不同 |

**切换矩阵示例**：

|  | GROUP_A | GROUP_B | GROUP_C |
|--|---------|---------|---------|
| GROUP_A | 0 | 30 | 60 |
| GROUP_B | 25 | 0 | 45 |
| GROUP_C | 55 | 40 | 0 |

表示：
- 从 GROUP_A 切换到 GROUP_B 需要 30 分钟
- 从 GROUP_B 切换到 GROUP_A 需要 25 分钟
- 从 GROUP_A 切换到 GROUP_C 需要 60 分钟

##### 排程引擎如何使用切换矩阵

```
当在同一资源上连续排两个任务时：
1. 获取前一任务的产品所属切换组
2. 获取后一任务的产品所属切换组
3. 根据资源/工作中心/全局优先级查找切换矩阵
4. 查询对应的切换时间
5. 在两个任务之间插入切换时间
6. 后一任务的开始时间 = 前一任务结束时间 + 切换时间
```

## 6. 核心业务逻辑

### 6.1 排程算法

#### 排程方向

| 方向 | 说明 |
|------|------|
| 正向排程 | 从最早开始时间向后排程，尽早开始生产 |
| 逆向排程 | 从交货期向前排程，尽可能晚开始以减少库存 |

#### 优先级规则

| 规则 | 说明 |
|------|------|
| EDD | 最早交货期优先（Earliest Due Date） |
| SPT | 最短加工时间优先（Shortest Processing Time） |
| FIFO | 先进先出（First In First Out） |

#### 有限产能排程

- 考虑资源每日可用产能（capacity_per_day）
- 当资源产能不足时，将工序安排到下一个可用时段
- 支持资源效率系数（efficiency）的调整

### 6.2 约束检测

| 约束类型 | 说明 |
|----------|------|
| 资源产能约束 | 检测资源是否超负荷 |
| 工序顺序约束 | 检测同一订单的工序是否按顺序执行 |
| 最早开始时间约束 | 检测是否早于订单的最早开始时间 |
| 交货期约束 | 检测预计完工时间是否晚于交货期 |

### 6.3 订单状态流转

```
计划订单创建 ──> 待排程(created)
                    │
                    ▼
               执行排程
                    │
                    ▼
             已排程(scheduled) ──> 转生产订单 ──> 生产订单(production)
                    │                                    │
                    ▼                                    ▼
             进行中(in_progress)                   进行中(in_progress)
                    │                                    │
                    ▼                                    ▼
             已完成(completed)                    已完成(completed)
```

### 6.4 工序状态流转

```
创建工序 ──> 待排程(pending)
                │
                ▼
           执行排程
                │
                ▼
         已排程(scheduled)
                │
                ▼
         进行中(in_progress)
                │
                ▼
         已完成(completed)
```

### 8.2 交货期默认时间

- 新建订单时，交货期默认时间为 **23:59**
- 这意味着订单需要在交货期当天23:59之前完成

### 8.3 延期判断规则

1. **已排程订单**：预计完工时间 > 交货期 → 延期
2. **未排程订单**：当前日期 > 交货期日期 且 订单未完成 → 延期

### 8.4 预计完工时间计算

预计完工时间 = 订单最后一道工序的 `scheduled_end`

如果工序没有排程时间，则使用订单的 `confirmed_end`（仅生产订单有效）

### 8.5 常见问题

**Q: 为什么订单显示为"待排程"但订单详情中有计划时间？**

A: 订单的显示状态是基于工序状态判断的。如果任何一个工序的状态为"pending"，则订单显示为"待排程"。这可能是因为排程只完成了部分工序。

**Q: 如何将计划订单转为生产订单？**

A: 在订单列表中，已排程的计划订单会显示"转生产"按钮。点击后可以选择确认时间或使用系统计算的排程时间。

**Q: KPI仪表板的数据多久更新一次？**

A: KPI数据在每次访问仪表板页面时实时计算，也可以点击"刷新数据"按钮手动刷新。

**Q: 排程结果可以撤销吗？**

A: 是的。排程结果会先保存在缓存中（预览模式），您可以选择"保存计划"确认排程结果，或选择"丢弃计划"撤销更改。
